## JSON 说明文件

每个图片文件都配有对应的 `.json` 文件，用于记录该图片被哪些文件引用。

### JSON 格式

```json
{
  "used_in": [
    "文件路径1",
    "文件路径2"
  ]
}
```

### 示例

**logo.jpg.json**
```json
{
  "used_in": [
    "README.md"
  ]
}
```

**计科.png.json**
```json
{
  "used_in": [
    "校园生活/培养方案.md"
  ]
}
```

**前端-image01.png.json**（未使用的图片）
```json
{
  "used_in": []
}
```

## 使用说明

### 引用图片

在 Markdown 文件中引用图片时，请使用绝对路径：

```markdown
![图片描述](/image/图片文件名)
```

例如：
```markdown
![Logo](/image/logo.jpg)
![计科开课规划](/image/计科.png)
```

### 添加新图片

1. 将图片文件放入本文件夹
2. 创建同名的 `.json` 文件
3. 在 `used_in` 数组中添加引用该图片的文件路径（相对于根目录）
4. 在相应的 Markdown 文件中使用绝对路径引用

### 更新图片引用

当图片被新的文件引用时：
1. 找到对应的 `.json` 文件
2. 在 `used_in` 数组中添加新的文件路径

当图片不再被某个文件引用时：
1. 找到对应的 `.json` 文件
2. 从 `used_in` 数组中删除该文件路径

当图片被删除时：
1. 同时删除对应的 `.json` 文件

## 图片命名建议

- 使用有意义的文件名（中文或英文）
- 避免使用特殊字符
- 建议格式：`描述性名称.扩展名`
- 例如：`2023版培养方案-计科.png`

## 注意事项

1. **路径统一**：所有图片引用都使用 `/image/` 开头的绝对路径
2. **同步更新**：修改或删除图片时，同步更新对应的 `.json` 文件
3. **文件清理**：定期检查 `used_in` 为空数组的图片，确认是否需要删除
4. **版本控制**：图片文件会占用较大的仓库空间，建议控制图片大小
5. **路径格式**：JSON 中的路径都是相对于项目根目录的路径

## 图片引用检查

如需手动检查哪些文件引用了特定图片：

```bash
grep -r "图片文件名" --include="*.md"
```
