## 💻 编辑器 vs IDE

### 文本编辑器

**定义**：专门用于编辑文本的软件

**特点：**
- 轻量级，启动快
- 功能单一，专注于编辑
- 高度可定制
- 需要手动配置环境

**常见编辑器：**

| 编辑器 | 特点 | 适用场景 | 价格 |
|--------|------|---------|------|
| **VS Code** | 微软出品，插件丰富，流行 | 全栈开发 | 免费 |
| **Sublime Text** | 轻量快速，响应快 | 轻量编辑 | 免费（部分功能收费） |
| **Notepad++** | Windows经典，轻量 | Windows简单编辑 | 免费 |
| **Vim** | 键盘操作，效率高 | Linux服务器 | 免费 |
| **Emacs** | 强大可编程 | 高级用户 | 免费 |

**编辑器适合：**
- 轻量级开发
- 快速查看和编辑文件
- 学习配置和自定义

### IDE（集成开发环境）

**定义**：包含全套开发工具的软件

**特点：**
- 重量级，功能全面
- 开箱即用
- 包含编辑器、编译器、调试器等
- 占用资源多

**常见IDE：**

| IDE | 特点 | 适用语言 | 价格 |
|-----|------|---------|------|
| **Visual Studio** | 微软出品，强大 | C#/.NET | 收费（社区版免费） |
| **IntelliJ IDEA** | 智能提示强大 | Java | 收费（社区版免费） |
| **PyCharm** | Python专用IDE | Python | 收费（社区版免费） |
| **WebStorm** | Web开发 | JS/前端 | 收费 |
| **Xcode** | 苹果官方IDE | Swift/iOS | 免费 |
| **Android Studio** | 安卓开发 | Android | 免费 |
| **Eclipse** | 老牌IDE | Java | 免费 |

**IDE适合：**
- 大型项目开发
- 需要完整工具链
- 企业级开发

### 如何选择？

**初学者：**
- 推荐：**VS Code**
- 原因：免费、插件多、文档丰富、社区活跃

**前端开发：**
- 编辑器：VS Code、WebStorm
- IDE：一般不用IDE

**后端开发：**
- Python：PyCharm 或 VS Code
- Java：IntelliJ IDEA
- C#：Visual Studio

**全栈开发：**
- 推荐：VS Code
- 原因：一个工具搞定所有语言

## 🔧 编译器和解释器

### 编译器

**定义**：将源代码转换为机器码的程序

**工作流程：**
```
源代码 (.c/.cpp)
    ↓ 编译
目标代码 (.o/.obj)
    ↓ 链接
可执行文件 (.exe/.out)
```

**特点：**
- 编译一次，多次运行
- 运行速度快
- 跨平台需要重新编译
- 错误检查严格

**常见编译型语言：**
- C
- C++
- Go
- Rust
- Java（特殊：编译成字节码）

**常见编译器：**
- **GCC**（GNU Compiler Collection）- 开源免费
- **Clang** - LLVM项目的一部分
- **MSVC**（Microsoft Visual C++）- Windows
- **javac** - Java编译器

**比喻：**
编译器就像**翻译官**，把源代码翻译成计算机能懂的机器码。

### 解释器

**定义**：逐行解释并执行源代码的程序

**工作流程：**
```
源代码 (.py/.js)
    ↓ 逐行解释
逐行执行
```

**特点：**
- 边解释边执行
- 运行速度较慢
- 跨平台容易
- 开发效率高

**常见解释型语言：**
- Python
- JavaScript
- Ruby
- PHP
- Shell脚本

**常见解释器：**
- **CPython** - Python官方解释器
- **Node.js** - JavaScript运行时
- **V8** - Chrome的JS引擎
- **Ruby MRI** - Ruby解释器

**比喻：**
解释器就像**同声传译**，边翻译边执行。

### 编译 vs 解释

| 特性 | 编译型 | 解释型 |
|------|-------|-------|
| **执行速度** | 快 | 慢 |
| **开发效率** | 低（需要编译） | 高（直接运行） |
| **跨平台** | 需要重新编译 | 通常跨平台 |
| **错误检查** | 编译时检查 | 运行时检查 |
| **例子** | C/C++、Go | Python、JavaScript |

## 🐙 版本控制系统

### Git

**定义**：分布式版本控制系统

**核心功能：**
- 记录代码历史
- 多人协作
- 版本回退
- 分支管理

**为什么重要：**
- 防止代码丢失
- 追踪修改历史
- 方便团队协作
- 开源项目必备

**基本概念：**
```
工作区（你看到的文件）
    ↓ git add
暂存区（临时存放）
    ↓ git commit
本地仓库（永久保存）
    ↓ git push
远程仓库（GitHub/Gitee）
```

**常用命令：**
- `git init` - 初始化仓库
- `git add` - 添加到暂存区
- `git commit` - 提交到本地仓库
- `git push` - 推送到远程
- `git pull` - 拉取远程更新

### GitHub / Gitee

**GitHub**：
- **定义**：代码托管平台（全球最大）
- **功能**：
  - Git远程仓库
  - 问题追踪
  - Pull Request
  - Actions CI/CD
- **优点**：全球最大的开源社区
- **缺点**：国内访问慢

**Gitee**（码云）：
- **定义**：国内代码托管平台
- **功能**：类似GitHub
- **优点**：国内访问快，免费私有仓库
- **缺点**：社区相对较小

### 其他版本控制工具

| 工具 | 类型 | 现状 | 说明 |
|------|------|------|------|
| **SVN** | 集中式 | 仍在使用 | 老企业项目常用 |
| **Mercurial** | 分布式 | 较少 | Linux内核曾用 |

## 📦 包管理器

### 为什么需要包管理器？

**手动管理依赖的痛点：**
- 下载麻烦
- 版本冲突
- 依赖关系复杂
- 更新困难

### 常见包管理器

#### Python - pip

**定义**：Python的包管理器

**基本用法：**
```bash
# 安装包
pip install pandas

# 安装指定版本
pip install pandas==1.5.0

# 卸载包
pip uninstall pandas

# 查看已安装的包
pip list

# 从requirements.txt安装
pip install -r requirements.txt
```

**镜像源（加速）：**
```bash
# 使用清华源
pip install -i https://pypi.tuna.tsinghua.edu.cn/simple pandas
```

#### JavaScript - npm

**定义**：Node.js的包管理器

**基本用法：**
```bash
# 初始化项目
npm init

# 安装包
npm install express

# 全局安装
npm install -g create-react-app

# 运行脚本
npm run dev
```

**国内加速：**
```bash
# 使用淘宝镜像
npm install -registry=https://registry.npmmirror.com
```

#### 其他包管理器

| 语言 | 包管理器 | 说明 |
|------|---------|------|
| **Java** | Maven, Gradle | 企业级项目 |
| **Go** | go mod | 内置在Go中 |
| **Ruby** | gem | RubyGems |
| **PHP** | Composer | PHP |
| **Rust** | cargo | 内置在Rust中 |
| **C++** | vcpkg, conan | 跨平台 |

## 🐳 容器和虚拟化

### Docker

**定义**：容器化平台

**核心概念：**
- **镜像**：软件模板
- **容器**：运行实例
- **Dockerfile**：构建脚本

**为什么重要：**
- 环境一致性
- 快速部署
- 资源隔离

**基本命令：**
```bash
# 运行容器
docker run nginx

# 查看容器
docker ps

# 停止容器
docker stop <容器ID>
```

### 虚拟机

**定义**：模拟完整计算机的软件

**常见虚拟机：**
- **VMware** - 商业软件
- **VirtualBox** - 开源免费
- **Hyper-V** - Windows自带

**容器 vs 虚拟机：**
- **容器**：轻量、启动快、共享内核
- **虚拟机**：重量级、启动慢、独立系统

### 浏览器开发者工具

**定义**：浏览器内置的调试工具

**打开方式：**
- Chrome: `F12` 或 `Ctrl+Shift+I`

**Python：**
- pdb（内置调试器）
- PyCharm调试器
- print()函数（最简单）

**JavaScript：**
- Chrome DevTools
- console.log()
- debugger关键字

**其他工具：**
- **Postman** - API测试
- **Wireshark** - 网络抓包
- **日志工具** - Logcat, Event Viewer

## ❓ 常见问题

### Q1: 新手推荐什么编辑器？
**A**:
- **强烈推荐：VS Code**
- 原因：
  - 免费
  - 跨平台
  - 插件丰富
  - 社区活跃
  - 文档丰富

### Q2: IDE和编辑器哪个好？
**A**:
- **新手**：编辑器（VS Code）
- **项目开发**：IDE（如PyCharm）
- **个人项目**：编辑器
- **企业项目**：IDE

### Q3: 编译器和解释器必须学吗？
**A**:
- **不需要深入**：了解概念即可
- **自动化**：IDE自动处理
- **进阶需要**：性能优化时需要了解

### Q4: Git和GitHub是什么关系？
**A**:
- **Git**：版本控制工具（软件）
- **GitHub**：代码托管平台（网站）
- **关系**：类似 银行卡 vs 银行

### Q5: Docker一定要学吗？
**A**:
- **新手**：不是必须的
- **就业**：很多公司要求会Docker
- **建议**：了解概念，简单使用
